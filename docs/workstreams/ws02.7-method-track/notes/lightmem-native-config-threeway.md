## 失配清单（只比较 a / b / c）

### LoCoMo

- answer/judge readout：paper 声明 GPT judge、GPT/Qwen/GLM backbone，但未给 answer
  prompt 与 LoCoMo 专用 judge prompt；复现目录给出 `ANSWER_PROMPT`、
  `ACCURACY_PROMPT`、temperature=0，而 repo method 默认完全不定义 answer/judge
  readout。（paper p.6, p.8, p.17, p.23-24；`search_locomo.py:258-280,456-463`；
  `llm_judge.py:22-74`；`configs/base.py:16-123`）
- compression/STM：paper Table 3 报告五组 `(r, th)`，复现 build 脚本当前实际
  `rate=0.6` 且 core STM 硬编码 512；repo 顶层默认 pre-compress OFF，组件被启用时
  rate=0.8。（paper p.8 Table 3；`add_locomo.py:157-174`；
  `memory/lightmem.py:171-174`；`configs/base.py:17-23`；
  `configs/pre_compressor/llmlingua_2.py:23-28`）
- topic：paper 明确启用 LLMLingua-2 + 512-token sensory buffer，但只用符号 τ；
  复现实际 core 给 τ 搜索 0.2→0.5，而 repo 顶层默认 topic OFF。（paper p.4, p.6；
  `factory/memory_buffer/sensory_memory.py:71-94`；`configs/base.py:25-35`）
- extraction/update/retrieval：复现目录是 extract=0.1、combined top-60、offline
  score=0.9；repo 默认是 extract=0.5、retrieve API top-10、offline score=0.9。
  paper 未声明这些数值。（`add_locomo.py:192,445-448`；`readme.md:89-97`；
  `configs/base.py:57-59`；`memory/lightmem.py:539-550,644-659`）
- embedding：paper 与复现目录都指 all-MiniLM-L6-v2；repo schema 只默认 HF backend，
  具体 model/dims 为 None，顶层 text embedder 也是 None。（paper p.17 Table 5；
  `readme.md:49-65`；`configs/text_embedder/base_config.py:4-10`；
  `configs/base.py:65-68`）

### LongMemEval

- backbone：paper Table 2 报告 GPT-4o-mini、Qwen3-30B-A3B-Instruct-2507、GLM-4.6；
  GPT 复现脚本与其一致，但 `run_lightmem_qwen.py` 当前实际是 `qwen-plus`，judge
  又是 `deepseek-chat`。repo method 默认不定义 answer/judge readout。（paper p.7
  Table 2；`run_lightmem_qwen.py:9-15`；`configs/base.py:16-123`）
- answer/judge 参数：复现脚本共用 0/2000/0.8；paper 未声明；repo memory-manager
  默认是 0.1/2000/0.1，但它不是 answer/judge client。（`run_lightmem_gpt.py:51-72`；
  `configs/memory_manager/base_config.py:8-18`）
- compression/STM：paper Table 2 报告九组 `(r, th)`；两个复现脚本都未传
  `compress_config`，因此实际 rate=0.8，且 core STM=512；repo 顶层默认 compressor
  OFF，启用组件时 rate=0.8。（paper p.7 Table 2；`run_lightmem_gpt.py:82-99`；
  `configs/pre_compressor/llmlingua_2.py:23-28`；`memory/lightmem.py:171-174`）
- extraction/retrieval/update：复现目录 extract=0.1、retrieve=20、独立 offline utility
  score=0.8；repo 默认 extract=0.5、retrieve=10、offline score=0.9。paper 未声明这些
  数值。（`run_lightmem_gpt.py:112,181`；`offline_update.py:39-42`；
  `configs/base.py:57-59`；`memory/lightmem.py:539-550,644-659`）
- 数据范围：Qwen 脚本把数据截成前 10 条，而论文 LongMemEval-S 描述为 500 个问题；
  GPT 脚本没有该截断。（`run_lightmem_qwen.py:157-159`；
  `run_lightmem_gpt.py:140,148`；paper p.20 §D.1）
- embedding：paper 与复现脚本均为 all-MiniLM-L6-v2；repo schema 没有具体默认
  model/dims。（paper p.17 Table 5；`run_lightmem_gpt.py:113-130`；
  `configs/text_embedder/base_config.py:4-10`）
